# 跨境电商分层AI客服系统

## 项目背景
本项目基于我正在经营的TikTok Shop马来西亚店铺真实业务场景设计。
作为无货源模式卖家，因语言障碍（马来语/英语）和无法24小时在线，
买家咨询经常无法及时得到回复，存在响应延迟和潜在信任流失风险。

## 真实案例
一名买家咨询世界杯周边商品能否在赛事前送达，因卖家未及时在线，
延迟12小时才收到回复。事后跟进发现：包裹最终正常按时签收，
说明及时的安抚式回复（即使无法给出精确到达日期）也能有效降低
买家焦虑，这成为本项目的核心设计依据。

## 核心设计：分层处理
不同类型的问题，AI采用不同处理策略：

| 问题类型 | 处理方式 |
|---------|---------|
| 客观信息（发货时效等） | AI直接准确回答 |
| 物流时效咨询 | AI安抚 + 说明不确定性，不承诺具体日期 |
| 退款/投诉/议价 | AI安抚 + 转交卖家处理，AI不做最终决定 |

## 为什么这样设计
AI如果直接承诺"具体哪天能到"，一旦物流延误，会造成比"暂时
没回复"更严重的信任损失。因此本系统的核心原则是：**可以不
完美，但不能说谎。**

## 技术栈
- Python
- Google Gemini API
- Prompt层面实现分层处理逻辑

## 如何运行
1. 安装依赖：`pip install` 相关库
2. 将代码中的 API_KEY 替换为你自己的 Gemini API Key
3. 运行 `python rag_crossborder.py`
